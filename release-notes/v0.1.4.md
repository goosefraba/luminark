# Luminark v0.1.4

## Fixed

- Repaired the packaged app bundle layout so release builds use a standard `Contents/Resources` resource location
- Re-signed packaged release apps after bundling so the shipped `.app` is structurally valid on disk
- Cleaned release zip output to avoid extra macOS metadata files

## Changed

- Added clearer installation guidance for the current unsigned, non-notarized release flow

## Assets

- Apple Silicon build: `Luminark-0.1.4-macos-arm64.zip`
- Intel build: `Luminark-0.1.4-macos-x86_64.zip`

## Notes

- These builds are still unsigned for Developer ID distribution and are not notarized yet
- On first launch, macOS may require `Open Anyway` or quarantine removal until notarized releases are in place
